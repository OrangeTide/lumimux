# libutf8

UTF-8 encoding/decoding and Unicode-version-aware character width lookup.

## Encoding and Decoding

`utf8_encode` and `utf8_decode` convert between Unicode codepoints and their
UTF-8 byte representations (up to 4 bytes per RFC 3629). Malformed input
produces the replacement character `U+FFFD`. `utf8_runelen` returns the byte
count needed for a given codepoint without writing anything.

## Character Width -- `rune_width`

`rune_width(cp)` returns the display width of a codepoint: 0 for combining
marks and default-ignorable format characters, 2 for East Asian wide/fullwidth
characters, 1 for everything else, and -1 for non-printable controls.

### Selecting a Unicode Version

The width tables are version-specific because the set of wide and zero-width
characters changes between Unicode releases. Call `rune_width_init()` once at
startup -- it reads the `UNICODE_VERSION` environment variable and selects the
matching built-in table. If the variable is unset or unrecognized, the latest
table is used. You can also switch at runtime with `rune_width_set("15.1.0")`.

### Supported Unicode Versions

- 9.0.0
- 12.1.0
- 15.1.0

### Table Generation

The file `width_tables.c` is generated by `gen-width-tables.sh` (in this
directory), which downloads `EastAsianWidth.txt`, `UnicodeData.txt`, and
`emoji-data.txt` from the [Unicode Character Database][1] for each version
and emits C arrays of `struct rune_range` used by the binary search in
`rune_width.c`.

To regenerate:

```sh
make gen-width-tables
```

[1]: https://www.unicode.org/Public
